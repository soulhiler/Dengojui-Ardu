## 1. Одноразовая настройка обновлений телефона (после неё — всё само)

Корень прошлых мучений: дашборд слушал только `127.0.0.1` — телефон
физически не мог скачать APK. Починено в этой ветке (бинд `0.0.0.0` +
mDNS-анонс «xiao-dash»).

```powershell
git fetch origin claude/repo-review-iuo7b5
git checkout claude/repo-review-iuo7b5   # или git pull, если уже на ней
pip install zeroconf                      # опционально: автопоиск ПК с телефона
# Скачать свежий APK из release apk-latest в dist\ :
#   https://github.com/soulhiler/Dengojui-Ardu/releases/tag/apk-latest
#   файл вида xiao-robot-v8-debug.apk → положить в dist\
py -3 tools\xiao_serial_telemetry.py      # дашборд: 0.0.0.0:8897 + mDNS xiao-dash
```

На телефоне (там пока v4): «Управление» → «Обновить» → вписать
`<IP_ПК>:8897` (IP — `ipconfig`). Это последний ручной ввод адреса:
начиная с v8 приложение после «Подключить» **само** находит ПК по mDNS и
предлагает установку, если в `dist\` лежит APK новее.

Цикл дальше: CI собрал → скачать `xiao-robot-vN-debug.apk` в `dist\` →
телефон сам предложит установку при следующем подключении.

## 2. Моторы «вообще не реагируют» — что уже известно и чем проверить

Прошивка: привод включён по умолчанию; рефлекс-safety — только бампер/УЗ
(оба отключены в `drive_config.h`), ToF на привод не влияет; во время
beep/melody моторы намеренно не крутят (звук теми же обмотками).
Тумблер «Привод» в v4 показывал «выкл» из-за бага приложения (читал
несуществующий ключ `ctrl_drive`; исправлено в v6 — ключ `drive_enabled`).

Диагностика (приложение v7+: на вкладке «Телеметрия» есть джойстик-оверлей):

1. Двигать джойстик и смотреть группу **«Привод»**: меняются ли
   `drive_cmd_l` / `drive_cmd_r`; рядом `drive_enabled`, `drive_watchdog`,
   `drive_safety` (в v6+ причина блокировки дублируется в статус-строке).
2. «Beep A» на «Управлении»: звук издают сами обмотки через TB6612 —
   пищит = силовая часть (TB6612, моторы, VM, GND) жива.
3. `drive_cmd_*` меняются, Beep пищит, колёса молчат → провода
   AO1/AO2/BO1/BO2 или механика.
   `drive_cmd_*` меняются, Beep молчит → питание VM (батарея!), STBY → 3.3V,
   общий GND — см. `docs/hardware/wiring-xiao-motor-tof.md`.
   `drive_cmd_*` не меняются → смотреть Serial платы
   (`py -3 tools/xiao_serial_telemetry.py --port COMx`) на приём /drive.
4. С ПК напрямую (плата в LAN): `http://<IP_платы>/drive?l=120&r=120`,
   затем `http://<IP_платы>/status` — видно cmd/watchdog без приложения.

Если дело в прошивке/PWM (камера занимает LEDC0; drivePwmInit ставит
канал на пины 5/6) — у Claude Code на ПК есть USB: прошивка, Serial-логи,
эксперименты на живой плате.

## 3. Прочее состояние

- Прошивку перезаливать не обязательно (1.3.0 build 16 уже стоит), но в
  ветке есть невлитый фикс Serial (`txTimeout 0→20 мс`, коммит fd35e29) —
  при следующей прошивке подхватится.
- ToF VL53L7CX: подключение — `docs/hardware/wiring-xiao-motor-tof.md`
  (**LPn → 3.3V обязательно**); библиотека
  `arduino-cli lib install "STM32duino VL53L7CX"`; проверка: `tof_ok=1` в
  телеметрии, сетка зон — `GET /tof`.
- PR #2 зелёный и без конфликтов; после проверки моторов на железе —
  можно переводить из драфта и мержить в `main`.
