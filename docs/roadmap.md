# План работ «Деньгожуй» (роадмап, 2026-06)

> Сводный план: где робот сейчас и что дальше. Связывает прошивку, spatial-модель,
> ориентацию и Wi-Fi-локализацию. Источник истины по приоритетам; обновлять по вехам.
> Детали — в `docs/hardware/*` и `spatial/`.

## Где мы сейчас (готово)

- **Прошивка** XIAO ESP32-S3: камера (`/stream` :82, `/capture`), ToF VL53L7CX
  (`/tof` сетка, `/scan360`, калибровка пола `/floorcal`, фильтр sigma, LPn на GPIO7),
  привод, мик, телеметрия, OTA. Камера+ToF+привод работают одновременно (fw 1.3.x).
- **Android-приложение** v9: вкладки Робот/Телеметрия/Управление, mDNS-автопоиск,
  самообновление по LAN, джойстик-оверлей.
- **Spatial**: автономный сервис мировой модели (`spatial/world_service.py` +
  `world_model.py`): сам тянет кадры, копит воксельную модель с уверенностью
  (log-odds), сохраняет на диск, переживает рестарт; 3D-визуализатор в дашборде.
  Этап 1 (камера+ToF → цветное облако) готов.
- **Исследования** в `docs/hardware/`: VL53L7CX, ориентация+карты, SBC (Orange Pi),
  лидары, камера+ToF→модель, Wi-Fi-локализация.

## Ближайшее железо в наличии

- **MPU6050 (GY-521)** — есть, ~$2. 6-осевой IMU (без магнитометра). Делит I2C-шину
  с ToF (адреса 0x68 vs 0x29, без конфликта). Это бюджетный источник курса.

## Очередь работ (по приоритету)

### 1. 🔴 Курс от MPU6050 → честная одометрия (разблокирует всё)
Без курса модель копится из одной точки. Шаги:
- Прошивка: чтение MPU6050 по I2C, отдавать gyro/accel(+таймстамп) в `/telemetry`
  (фьюз на сервере) **или** Mahony/комплементарный фильтр на плате → yaw.
- `spatial/world_service.py::PoseEstimator`: добавить источник курса (yaw из IMU),
  связка с энкодерами (гиро сквозь слип, энкодеры бьют дрейф).
- Результат: spatial-модель строится со склейкой по позе; spin/turntable перестаёт «плыть».
- Док: `docs/hardware/orientation-and-mapping-decisions.md`.

### 2. 🟠 Плотная глубина (этап 2 spatial)
- Сервер: Depth Anything (моно-глубина) + ToF для метрического масштаба (аффинный
  МНК) → плотное цветное облако вместо 64 точек/кадр.
- Перф: на сервере с GPU / Jetson Orin (не на Orange Pi 5 NPU — трансформер сломан).
- Док: `docs/hardware/camera-tof-spatial-model.md`.

### 3. 🟠 Wi-Fi-якорь позиции (без покупок) → сброс дрейфа
- Прошивка: эндпоинт `/wifiscan` (вектор RSSI всех AP, асинхронно — скан роняет связь).
- Сервер: авто-сбор RSSI-отпечатка **в координатах world_model**; матчинг **SE-WKNN**
  (ранговая корреляция); частичный фильтр с **ограничениями по occupancy-grid**;
  фьюз с одометрией+IMU. Ожидаемо ~1–2 м, лечит дрейф / «kidnapped».
- Док: `docs/hardware/wifi-localization-method.md`.

### 4. 🟡 Персистентная карта для навигации (этап 3) — нужен SBC
- RTAB-Map (RGB-D из этапа 2 + одометрия+IMU) → `.db` на сервере → робот
  переоткрывает в режиме локализации; мульти-сессия дополняет.
- Железо: 4-ГБ SBC (Orange Pi 5 / Pi 4-5). Опц. дешёвый 360° LiDAR (LD06/LD19) для
  надёжной глобальной карты.
- Доки: `orientation-and-mapping-decisions.md`, `lidar-landscape-2026.md`, `orange-pi-lineup-2026.md`.

### 5. 🟢 Автономность (brain/) поверх модели
- Реактивный объезд по occupancy-grid + cliff (уже есть в ToF), затем планирование
  пути по карте, патрулирование точек. Опираться на `brain/` (SafetyGovernor).

## Принципы
- **XIAO = источник данных, сервер/SBC = вычисления** (модель/SLAM/ML).
- **Курс — IMU+энкодеры; Wi-Fi — якорь позиции против дрейфа** (не компас).
- Каждый шаг — на имеющемся железе, где можно; покупки только когда уперлись.
- Прошивку проверять на железе (CI собирает, но реальность — на плате).
