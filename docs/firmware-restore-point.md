## Откат на 1.4.1 (если 1.4.3 плохо себя ведёт)

Откатывается **только прошивка** (каталог `xiao_cam_stream/`), остальной репозиторий не трогаем:

```bash
# 1. Достать исходник прошивки 1.4.1 в рабочее дерево
git fetch origin restore/fw-1.4.1
git checkout origin/restore/fw-1.4.1 -- xiao_cam_stream/

# 2. Перепрошить (Windows-скрипт проекта; сам найдёт XIAO по USB, PSRAM=OPI)
.\tools\xiao_flash_and_telemetry.ps1
#   либо вручную:
arduino-cli compile --fqbn esp32:esp32:XIAO_ESP32S3:PSRAM=opi xiao_cam_stream
arduino-cli upload  --fqbn esp32:esp32:XIAO_ESP32S3:PSRAM=opi -p <COM> xiao_cam_stream

# 3. Вернуть рабочее дерево к актуальному состоянию, когда откат больше не нужен
git checkout HEAD -- xiao_cam_stream/
```

Проверка после отката: `GET /telemetry` → `fw_version` = `1.4.1`, `fw_build` = `24`.

## (Опц.) Бинарный бэкап ЖИВОЙ платы перед прошивкой

Канон бэкапа здесь — исходник на ref выше (бинарники билда в `.gitignore`).
Но если хочется снять точную копию того, что **прямо сейчас** во флеше платы
(на случай, если 1.4.1 пересобрать не выйдет 1-в-1) — это делается на твоей
машине, не из облака:

```bash
# дамп всех 8 МБ флеша XIAO ESP32-S3 в файл (esptool из ядра esp32)
esptool --chip esp32s3 -p <COM> -b 460800 read_flash 0 0x800000 xiao-fw-1.4.1-backup.bin
# восстановление при необходимости:
esptool --chip esp32s3 -p <COM> -b 460800 write_flash 0 xiao-fw-1.4.1-backup.bin
```

Файл `*.bin` держать вне репозитория (он большой и не версионируется);
положить в `dist/` или личное хранилище.

## Прошивка 1.4.3 (на что смотреть после заливки)

- `GET /telemetry` → `fw_version` = `1.4.3`, `fw_build` = `26`.
- Новое: `drive_front_mm` (фронтальная дистанция) и `drive_gov` (масштаб
  скорости %, 100 = не режет).
- Поведение: при движении **вперёд** к стене ход плавно сбавляется (ближе
  150 мм — стоп вперёд, дальше 600 мм — полный газ). Реверс/повороты не
  затронуты. Пороги — `XIAO_DRIVE_GOV_STOP_MM` / `SLOW_MM` в `drive_config.h`.
- Выключить регулятор: `XIAO_DRIVE_TOF_GOV 0` в `drive_config.h` и перепрошить.
