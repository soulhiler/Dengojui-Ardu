- `arduino-cli` 1.5.0 (portable в `tools/`, gitignored); переиспользовано уже

- `arduino-cli compile --fqbn esp32:esp32:XIAO_ESP32S3:PSRAM=opi xiao_cam_stream`
  → **успех, 0 ошибок, 0 предупреждений** в наших источниках.
- Размер: 1 181 569 Б флэш (**35 %**), глобальные 62 476 Б (**19 %**).
- `secrets.h` создан из example (gitignored, не коммитится).

### Что это даёт
Прошивочные правки **Фаз 1–3 подтверждены на уровне компиляции**: токен/
`apiAuthOk`, `initCamera` retry + `cam_fail`, `telemetryMicApplyState`
(I2S-release), антидребезг энкодеров + рефлекс безопасности + `spd_*`/
`drive_safety` в `/telemetry`. Это снимает прежнюю оговорку «в этой среде
собрать нельзя» для прошивки. Runtime/HW (cross-core I2S, рефлекс,
одометрия, реальные FPS) — по-прежнему требует платы.

Транспортно-нейтрален (ПК-сервер или порт на телефон).

- `robot.py` — клиент платы (`/telemetry`, `/capture`, `/drive`, `?t=`).
- `perception.py` — `Recognizer` + `Dummy` (без зависимостей) + `Brightness`

  архитектура от модели не зависит.
- `safety.py` — `SafetyGovernor` (чистая функция `decide`): рефлекс прошивки
  приоритетен, УЗ-стоп (`us_cm==0`≠препятствие), таймаут перцепции,
  неуверенность → стоп; темп команд `min(0.25, watchdog/2)` < 450 мс.
- `brain.py` — CLI + цикл perceive→decide→govern→act; аварийный `stop()`
  при потере связи и в `finally`; `--dry-run/--once` для офлайн-проверки.
- `test_safety.py` — 9 юнит-тестов governor.
- `docs/brain-api.md` — контракт и модель безопасности; `brain/README.md`;
  обновлён корневой README.

## 2026-05-17 — Фаза 3 (сенсоры и базовая автономность)

Код сделан корректным и устойчивым, **по умолчанию инертен** (УЗ/бамперы
выключены пинами `0`). Физическая разводка, выбор GPIO и проверка на плате —
шаг пользователя (см. `drive_config.h.example`); собрать/протестировать в этой
среде нельзя.

- `drive_config.h(.example)`: добавлены `DRIVE_ENC_DEBOUNCE_US` (антидребезг
  энкодера, по умолч. 150 мкс) и `DRIVE_US_STOP_CM` (порог рефлекса по УЗ);
  в `xiao_drive.h` — защитные `#ifndef` дефолты.
- Энкодеры: антидребезг в ISR по `micros()` (отбрасывает слишком близкие
  фронты); добавлен расчёт скорости `spd_l/spd_r` (тики/с).
- Рефлекс безопасности: единый латч (бампер | близкое УЗ), перекрывает
  команды клиента через гейт в `driveApplyMotors` (рефлекс важнее клиента),
  авто-сброс при пропадании препятствия. `us_cm==0` (нет эха/вне диапазона)
  НЕ считается препятствием — нет ложного стопа.
- УЗ читается до оценки безопасности; `pulseIn` троттлится ~120 мс.
- `/telemetry`: добавлены `spd_l`, `spd_r`, `drive_safety`.
- Android `TelemetryPoller`+`MainActivity`: компактная сводка сенсоров
  (СТОП-рефлекс/бампер/УЗ см/enc/v) в строке статуса.
