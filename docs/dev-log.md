### Фаза 2 — Надёжность ядра (готово)
- Android `MicPlayer`: bounded connect, авто-reconnect с backoff 0.8→4 с,
  корректный `stop()`, статус отражает реальное состояние.
- Android `MjpegStream`: конечный `readTimeout` 6 с (не виснет, реконнект);
  при переполнении буфера сохраняем хвост от последнего SOI; убран дубль-импорт.
- Android `DriveClient`: ошибки сети/HTTP-код выводятся в статус по смене
  состояния (видно «нет связи» / 403 неверный токен), без спама на 120 мс.
  `TelemetryPoller` намеренно тихий (его сбой виден как устаревший Wi-Fi).
- Поворот экрана: уже покрыт манифестом (`portrait` + `configChanges`) —
  кода не требуется.
- Прошивка: `initCamera()` — 3 попытки + `esp_camera_deinit` + восстановление
  статус-LED на провале; счётчик `cam_fail` в `/telemetry`.
- Прошивка: освобождение I2S при `/control?mic=0` (`telemetryMicApplyState`,
  безопасно к Core 1 `micTcpTask` через гейты `gCtrlMicEnabled`/`gMicOk`).
