# Моторы с энкодерами для «Деньгожуй» (2026)

> Выбор мотор-редукторов с квадратурными энкодерами под текущую прошивку (`drive_config.h`
> рассчитан на AB-энкодеры `DRIVE_ENC_*_A/B`, привод TB6612, дифференциальный 4WD/skid-steer).
> Итог: брать **TT Motor с AB-энкодером (чёрный, 1:90 / 175 об/мин)**, 2 шт. с энкодером на 4WD.

## Зачем именно энкодерные

Прошивка считает **квадратуру A/B** → скорость **и направление**. Поэтому:
- ❌ обычный мотор + щелевой ИК-диск (LM393) = один канал, без направления — не подходит.
- ✅ нужен мотор с **холловским AB-энкодером** (6 проводов: M+, M−, VCC, GND, A, B).

Показанные в магазине обычные TT/JB (JB207/JB209, жёлтый MT048, чёрный 1:48) — **без энкодеров**.

## Сравнение двух энкодерных вариантов

| | Вариант 1 (синий, метал 110rpm) | **Вариант 2 (чёрный TT Encoder)** ✅ |
|---|---|---|
| Энкодер AB | да, **PPR не указан** (обычно низкий) | да, **2 496 PPR (1:45) / 4 680 PPR (1:90)** на оборот колеса |
| Разъём | голые 6 проводов | **JST PH2.0 6-pin + кабель** (plug-and-play) |
| Шестерни | полный металл | полуметалл (металл+пластик) |
| Напряжение мотора | 6–12 В | **3–12 В** |
| Питание Холла | 3.3–5 В | 3.3–5 В |
| Вал | двухвальный (แกนคู่) | одновальный |
| Документация PPR | нет | есть (Yahboom encoder) |
| Цена | ฿208 | ฿279 |

**Выбор: Вариант 2 (чёрный), вариация 1:90 / 175 об/мин / 4 680 PPR** — высокое и документированное
разрешение энкодера (точная одометрия под будущий SLAM) + готовый разъём. 1:45/350 об/мин — только
если нужна скорость в ущерб моменту/точности.

## Спецификация выбранного (чёрный TT Encoder, 1:90)

| Параметр | Значение |
|---|---|
| Передаточное / скорость | 1:90 / 175 об/мин @6 В |
| Момент | ~2.4 kgf·cm |
| Напряжение мотора | 3–12 В (на 7.4 В 2S работает напрямую) |
| Ток | без нагрузки ≤200 мА, **стопор ~1.2 А** |
| Энкодер | магнитный Холл, AB-квадратура, square wave |
| Питание энкодера | 3.3–5 В (→ ESP32 напрямую) |
| Разрешение | **4 680 PPR** на оборот выходного вала |
| Разъём | JST PH2.0 6-pin (кабель в комплекте) |

## Совместимость и распиновка (под `drive_config.h`, без правок прошивки)

- **3–12 В мотор** → VM TB6612 = аккумулятор 7.4 В (отдельный buck на моторы не нужен).
- **Энкодер 3.3–5 В** → прямо в ESP32 (без преобразователя уровней).
- **AB-квадратура** → совпадает с `DRIVE_ENC_*_A/B`.

```
Encoder VCC → 3.3 В (ESP32 3V3)        Encoder GND → GND (общая)
Phase A (жёлтый) → D6/GPIO43 (лев.) , D8/GPIO7  (прав.)
Phase B (синий)  → D7/GPIO44 (лев.) , D9/GPIO8  (прав.)
Motor + (красный) / Motor − (чёрный) → TB6612 AO1/AO2 (лев.) , BO1/BO2 (прав.)
```
Колесо крутится «не туда» → поменять M+/M− (направление) или A/B (знак счёта).

## Сколько брать на 4 мотора (4WD)

Привод **дифференциальный (skid-steer)**: оба колеса борта вращаются синхронно, поэтому одометрии
хватает **одного энкодера на сторону** → **2 энкодерных мотора (1 левый + 1 правый)**, остальные 2 —
обычные. 4 энкодера нужны только при независимом управлении каждым колесом (меканум/полный 4WD), чего
в нашей прошивке нет.

⚠️ **Все 4 мотора на бортах должны быть одинаковые по об/мин** — два мотора одной стороны работают
параллельно на одном канале; разные модели «дерутся» (один тянет, другой тормозит). Идеально — все 4
одной модели (TT 1:90), из них 2 с энкодером + 2 без.

## ⚠️ Ток и драйвер при 4 моторах

2 мотора на канал = ток складывается. TT-мотор стопор ~1.2 А ×2 = ~2.4 А, а **TB6612 держит ~1.2 А
непрерывно на канал** (3.2 А пик) → один TB6612 на 4 мотора **на пределе**.

**Решение без новых пинов:** поставить **второй TB6612**, а его 6 логических входов завести
**параллельно** первому (те же D0–D5). Тогда каждый драйвер крутит по 2 мотора (один на сторону),
ток делится между драйверами, а **GPIO не добавляется** (важно — на XIAO свободных пинов нет, см.
[pinout-wiring-2026.md](pinout-wiring-2026.md)). Альтернатива — более мощный драйвер (напр. сдвоенный).

## Открытые вопросы (уточнить перед покупкой)

- **Модель/обороты текущих 4 моторов** — чтобы решить: докупить 2 энкодерных совпадающих или менять
  весь комплект на одинаковые.
- **Точный PPR/число полюсов** энкодера — для пересчёта тиков на оборот колеса в одометрии.
- Нужен ли **второй TB6612** (по току) — см. выше.
