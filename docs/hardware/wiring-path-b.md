# Схема подключения (Путь B — гибрид): XIAO ESP32-S3 + Raspberry Pi + сенсоры

> Конкретная разводка рекомендованного гибрида: **XIAO остаётся на моторике без переразводки**,
> а сенсоры/зрение садятся на Raspberry Pi. Основано на анализе пинов
> ([pinout-wiring-2026.md](pinout-wiring-2026.md)) и наборе сенсоров
> ([sensors-2026.md](sensors-2026.md)). Визуальная схема — [diagrams/pinout-path-b.svg](diagrams/pinout-path-b.svg).

## Блок-схема системы

```
   ┌──────────────────┐
   │ Аккумулятор 2S   │ 7.4 В (2× 18650 Li-ion)
   │ Li-ion + защита  │
   └───┬──────────┬───┘
       │ VM       │ Vin
       │      ┌───▼────────────┐
       │      │ DC-DC buck → 5В│  (≥3A, запас под Pi+LiDAR)
       │      └──┬──────────┬──┘
       │      5В │       5В │
   ┌───▼─────────▼──┐   ┌───▼──────────────┐
   │  TB6612FNG     │   │ Raspberry Pi Zero │
   │  драйвер мотор.│   │ 2 W  (+CSI камера)│
   └─┬───┬──────┬───┘   └─┬───┬────┬───┬────┘
  Лев │   │Прав │ 6 лог.  │I²C│SPI │UART│USB
   M ◄┘   └► M  │ сигналов │   │    │    │
               ▲          ▼   ▼    ▼    ▼
        ┌──────┴──────┐  [TCA9548A→ToF] [BNO085] [LiDAR/GPS] [OAK-D/depth]
        │ XIAO ESP32  │  [INA226]
        │ -S3 Sense   │  [BME280][BH1750]
        │ (камера,мик)│
        └──────┬──────┘
       4 энкод.│  ▲  Wi-Fi (этап 1)  /  USB-serial (этап 2)
        ◄──────┘  └──────────────────────────────►  Raspberry Pi
```

Логика: **Pi** — высокий уровень (зрение, сенсоры, навигация, стрим) ↔ **XIAO** — реальное время
(моторы, энкодеры, watchdog). Связь Wi-Fi (этап 1) или USB-serial (этап 2).

## 1. XIAO ESP32-S3 — без изменений (как в `drive_config.h`)

| Пэд | GPIO | → Куда | Назначение |
|---|---|---|---|
| D0 | 1 | TB6612 **AIN1** | направление левого |
| D1 | 2 | TB6612 **AIN2** | направление левого |
| D2 | 3 | TB6612 **BIN1** | направление правого |
| D3 | 4 | TB6612 **BIN2** | направление правого |
| D4 | 5 | TB6612 **PWMB** | ШИМ правого (20 кГц) |
| D5 | 6 | TB6612 **PWMA** | ШИМ левого (20 кГц) |
| D6 | 43 | Энкодер L канал A | ← с энкодера левого мотора |
| D7 | 44 | Энкодер L канал B | ← |
| D8 | 7 | Энкодер R канал A | ← с энкодера правого мотора |
| D9 | 8 | Энкодер R канал B | ← |
| D10 | 9 | (опц.) бампер | микропереключатель → GND, `INPUT_PULLUP` |
| — | 5V | питание платы | от 5В-рейла или USB от Pi |
| — | 3V3 | → TB6612 VCC/STBY | логическое питание драйвера |
| — | GND | общая земля | ⚠️ обязательно общая с Pi и драйвером |

Камера/мик — внутренний B2B Sense (GPIO 10–18, 38–48, 41/42), на гребёнку не выведены.

## 2. TB6612FNG — драйвер моторов

| Вывод TB6612 | → Куда |
|---|---|
| VM | + аккумулятора моторов (6–12 В) |
| VCC | 3.3 В от XIAO (логика) |
| STBY | 3.3 В (или GPIO; в проекте закорочен на 3.3 В) |
| GND | общая земля |
| PWMA / AIN1 / AIN2 | D5 / D0 / D1 XIAO |
| PWMB / BIN1 / BIN2 | D4 / D2 / D3 XIAO |
| AO1, AO2 | левый мотор |
| BO1, BO2 | правый мотор |

## 3. Raspberry Pi — сенсоры по шинам (40-pin)

| Шина / пин Pi (BCM) | Сенсор(ы) | Заметка |
|---|---|---|
| **I²C-1** SDA=GPIO2, SCL=GPIO3 | **TCA9548A** (мультиплексор) → VL53L5CX (фронт), 2× VL53L1X (бока), 2× VL53L0X (вниз, cliff); **INA226** (батарея); **BME280**; **BH1750** | ToF-семейство 0x29 → через мультиплексор; остальные адресуются прямо |
| **SPI-0** MOSI=10, MISO=9, SCLK=11, CE0=8 | **BNO085** (IMU) | по SPI (на ESP32 был бы конфликт, на Pi — ок); альт. UART-RVC |
| **I²S** BCLK=GPIO18, LRCLK=GPIO19, DOUT=GPIO21 | **MAX98357A** (усилитель + динамик 4–8 Ω/3 Вт) | аудио-**выход** (TTS/звуки); Vin 5 В; SD не трогать = моно; см. §3a |
| **UART** (напр. `/dev/ttyAMA*`) TXD=14, RXD=15 | **RPLIDAR C1 / LD19** | LiDAR шлёт поток → нужен RX Pi; питание 5 В |
| **UART2** (доп., напр. GPIO0/1 или USB-UART) | **GPS NEO-M8N** | NMEA, RX Pi |
| **USB** | **OAK-D Lite / RealSense** (опц., depth) + линк к XIAO (этап 2) | depth-камеры требуют USB |
| **CSI** | камера Pi | основной обзор/CV |
| свободные GPIO | бампер, XSHUT-линии ToF | XSHUT нужен, если ToF без мультиплексора |
| 3V3 / 5V / GND | питание датчиков | I²C-датчики 3.3 В; LiDAR — 5 В |

### I²C-карта адресов на шине Pi
| Адрес | Устройство |
|---|---|
| 0x70 | TCA9548A (за ним все ToF 0x29 по каналам) |
| 0x40–0x4F | INA226 (выбор A0/A1) |
| 0x4A/0x4B | BNO085 — *если по I²C* (рекомендуем SPI/UART) |
| 0x76/0x77 | BME280 |
| 0x23/0x5C | BH1750 |

### 3a. Аудио: вход на ESP32, выход на Pi

Разделение по «мозгам»: **микрофон (вход) — на ESP32** (PDM, уже работает, стрим PCM по TCP :81),
**динамик (выход) — на Pi** через I²S-усилитель **MAX98357A** (робот «говорит» через TTS/файлы).

| Вывод MAX98357A | → Pi (BCM) | Заметка |
|---|---|---|
| Vin | 5 В (pin 2/4) | 2.5–5.5 В; от 5 В громче |
| GND | GND | общая земля |
| BCLK | GPIO18 (pin 12) | I²S битовый такт |
| LRC | GPIO19 (pin 35) | I²S word select |
| DIN | GPIO21 (pin 40) | данные с Pi → вход усилителя |
| SD | не трогать | подтяжка = моно (L+R)/2; на GND = выключение |
| GAIN | не трогать | по умолчанию 9 дБ |
| Динамик +/− | зелёный клеммник | 4–8 Ω, до 3 Вт |

Софт на Pi: включить I²S-оверлей в `/boot/config.txt` (`dtoverlay=googlevoicehat-soundcard` или
профиль max98357a), затем `aplay` / TTS (piper, espeak). На XIAO усилитель **не вешаем** — там нет
3 свободных пинов под I²S.

## 4. Питание

```
2S Li-ion 7.4В ─┬─► VM TB6612 (моторы напрямую)
                └─► DC-DC buck 5В (≥3A) ─┬─► Pi Zero 2 W (5V)
                                          ├─► XIAO 5V  (или USB от Pi)
                                          ├─► LiDAR 5V, depth-камера (USB Pi)
                                          └─► MAX98357A Vin 5В (усилитель)
   3.3В:  XIAO 3V3 → TB6612 VCC/STBY;  Pi 3V3 → I²C-датчики (ToF/IMU/INA/BME/BH1750)
   GND:   ЕДИНАЯ общая земля для всех узлов (аккумулятор, драйвер, XIAO, Pi, сенсоры)
```

- **Pi Zero 2 W** в пике до ~0.5 А @5 В; LiDAR-моторчик — сотни мА → buck с запасом.
- Моторное питание (VM) держать отдельно от логики/Pi-рейла, но **GND общая**.
- 5 В сенсоры (HC-SR04, US-100) — делитель на линиях, идущих в 3.3 В входы (ECHO).

## 5. Линк XIAO ↔ Pi

- **Этап 1 (Wi-Fi):** провода нет. Pi ходит к XIAO по HTTP (`/drive`, `/telemetry`) и TCP :81 (мик),
  как `tools/xiao_cam_proxy.py`. Оба в одной Wi-Fi.
- **Этап 2 (USB-serial):** USB-кабель Pi↔XIAO (данные + питание XIAO). Команды привода строками
  по serial (требует флага `XIAO_SERIAL_DRIVE` в прошивке); телеметрию XIAO уже печатает в Serial.
