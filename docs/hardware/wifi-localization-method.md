# Ориентация/локализация по Wi-Fi для робота (исследование 2026)

> Можно ли использовать встроенный Wi-Fi XIAO ESP32-S3 для ориентации робота:
> по сигналу до роутера (RSSI), по времени пролёта (FTM), по отражённому сигналу
> (CSI). Decision-oriented, с реальными цифрами и ESP32-спецификой.
>
> Сверено с ESP-IDF доками, IEEE/arXiv-замерами на ESP32, реальными проектами.
> 403-блокировки → часть из выжимок, несущее подтверждено ≥2 источниками.

## Главный вывод (рамка, которую нельзя обойти)

**Wi-Fi даёт ПОЗИЦИЮ, а не КУРС.** Направление (куда робот смотрит) из Wi-Fi
статически не вытащить: для угла прихода (AoA) нужна фазо-когерентная антенная
**решётка**, а у ESP32 **одна антенна** (эталон «видеть Wi-Fi» — ESPARGOS из
**8 синхронизированных** чипов). Поэтому:

- **Курс/ориентация — это работа IMU (MPU6050) + одометрии**, не Wi-Fi.
- Роль Wi-Fi у нас — **абсолютная позиция без дрейфа**: одометрия+IMU точны
  коротко, но накапливают ошибку; Wi-Fi даёт шумный, но **не уплывающий** фикс,
  который **сбрасывает накопленный дрейф** и отвечает «в какой я комнате» /
  восстанавливает позу после «похищения» (kidnapped robot).

«Направление» из Wi-Fi возможно лишь как **курс движения** (вектор смещения
между позициями во времени) — и то это не ориентация корпуса, а направление пути.

## Три режима — реальные цифры и ESP32-S3

| Режим | Что даёт | Точность (реально) | Железо | Вердикт под нас |
|-------|----------|--------------------|--------|-----------------|
| **RSSI** | сила сигнала AP → дистанция/отпечаток | трилатерация ~2 м (нужно ≥3 AP с известными коорд.); отпечаток «комната» ~75–100% | **0** (есть) | ✅ для грубого фикса/комнаты |
| **FTM** (802.11mc) | время пролёта → дистанция | ~1.5–3 м indoor после калибровки (~5 м сырой) | ESP32-S3 маяки | ⏳ апгрейд (расставить маяки) |
| **CSI** | фаза/амплитуда по поднесущим (мультипас) | суб-метр в лаборатории, но per-room ML + сервер | 0 (но AoA — нет) | ❌ кроличья нора для навигации |

### RSSI (по сигналу до роутера/AP)
- `WiFi.RSSI()` — связанный AP; `WiFi.scanNetworks()` → RSSI всех видимых (в
  квартире обычно **5–25 AP**). Скан **1.5–4 с** и **роняет связь** со станцией —
  делать асинхронно/«скан-потом-реконнект».
- Шум: ±10–20 дБ стоя на месте (одна антенна, мультипас) → «высокоточная оценка
  невозможна». Трилатерация ~2 м, **но дома обычно 1–2 своих AP** (< нужных 3 с
  известными координатами) → чистая трилатерация плохо подходит.
- **Отпечаток (fingerprinting)** надёжнее трилатерации в мультипасе: карта
  «координата → вектор RSSI», потом k-NN. Реальные ESP32-проекты: ~75–100% точность
  «в какой комнате» (ani8897 ~2 м; joaocarvalho ~75% по комнатам).

### FTM (по времени пролёта, 802.11mc)
- **ESP32-S3 поддерживает FTM** (классический ESP32 — НЕТ, это главная ловушка).
  API `esp_wifi_ftm`, примеры initiator/responder.
- **ESP↔ESP работает без FTM-роутера** (домашние роутеры FTM обычно не умеют):
  расставить **свои ESP32-S3 маяки** (SoftAP-responder) в известных точках, робот
  (initiator) меряет дистанцию к каждому → трилатерация. Espressif прямо советует
  ESP↔ESP (само-калибровка).
- Точность на ESP32 (peer-reviewed замер): открытое ~1 м; indoor 20 МГц ~40% <2 м,
  ~10% >8 м (мультипас); с on-device ML ~80% <1.5 м. Нужна калибровка нуля на маяк.
- Тоже **только дистанция, без курса**.

### CSI (по отражённому сигналу)
- ESP32 отдаёт CSI (`esp_wifi_set_csi`, тулкиты esp-csi / ESP32-CSI-Tool). Богаче
  RSSI — ловит мультипас/движение.
- **Реально на одном чипе:** присутствие/движение/дыхание («кто-то в комнате»).
  Локализация по CSI-отпечатку бьёт RSSI (суб-метр в статье), но требует **сервер +
  ML, тренировку под каждую комнату**, хрупко к перестановке мебели, фаза шумная.
- **AoA / «видеть отражения для ориентации» — невозможно на одной антенне**
  (ESPARGOS = 8 синхронных чипов; робо-CSI-SLAM как WiROS/ViWiD — на многоантенных
  картах). Идея «курс по отражённому сигналу на ESP32» — не нашлось ни одного
  рабочего проекта, спекулятивно.
- Вердикт: для навигации/ориентации — плохой ROI; sweet spot одного чипа —
  детектор движения, не навигация.

## 🎯 Предлагаемый метод под наш робот (без покупок)

**Само-строящийся Wi-Fi-отпечаток, привязанный к нашей мировой модели → сброс
дрейфа.** Изящно ложится на то, что у нас уже есть (`spatial/world_model.py` —
персистентная модель в координатах; робот ездит с одометрией):

1. **Сбор на ходу (бесплатно):** в каждой позе робот делает `scanNetworks()` и
   пишет **вектор RSSI всех видимых AP**, тегируя его **своей координатой из
   world_model**. Так радио-карта строится **в собственных координатах робота,
   без ручного обмера** — побочный продукт обычного исследования помещения.
2. **Использование:** позже робот читает RSSI и матчит k-NN против карты →
   грубая **абсолютная позиция** (в координатах его же модели).
3. **Сплав:** одометрия+IMU(MPU6050) ведут позу коротко и дают **курс**; Wi-Fi-фикс
   раз в N секунд **ограничивает дрейф** и восстанавливает позу после потери
   (kidnapped/рестарт). Wi-Fi → поправка позиции в `PoseEstimator`, не курса.

Это даёт «грубый GPS внутри дома» **на уже имеющемся железе**, ровно для того,
чего не хватает одометрии — нескользящего абсолютного якоря. Точность комната/
пара метров — но именно для сброса дрейфа этого достаточно.

**Где жить коду:** скан и RSSI-вектор — на XIAO (отдавать в `/telemetry` или
новый `/wifiscan`); карта и k-NN — на сервере в `PoseEstimator` (в духе
«XIAO=источник, сервер=счёт»). Скан роняет связь → делать редко/асинхронно, или
на стоянках.

## Лестница апгрейда

1. **Сейчас:** RSSI-отпечаток ↔ world_model (метод выше) — 0 ₽, грубый сброс дрейфа.
2. **Точнее (~2 м):** 3–4 **ESP32-S3 FTM-маяка** в известных точках → трилатерация.
   Стоит ~$5–10 и развёртывания, но даёт стабильный метрический якорь.
3. **Не тратить силы:** CSI-AoA/«ориентация по отражениям» на одной антенне — тупик;
   CSI брать только если захочется **детектор присутствия** (другая задача).

## Итог одной строкой

Wi-Fi — это **якорь позиции против дрейфа**, не компас. Курс оставляем IMU+энкодерам;
Wi-Fi (сперва бесплатный RSSI-отпечаток в координатах нашей модели, потом опц.
FTM-маяки) сбрасывает накопленную ошибку и отвечает «где я в доме».
