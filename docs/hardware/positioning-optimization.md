# Оптимизация позиционирования и стройки карты мира

Глубокий разбор сессии автономного скана (2026-06-16) + исследование методов +
что сделано и что осталось. Контекст: XIAO ESP32-S3 + VL53L7CX (ToF) + MPU6050
(IMU курса), turntable-режим (энкодеров нет → отслеживается только поворот).

## 1. Что показала сессия (данные)

Два автономных спин-скана (`tools/spin_scan.py`, робот сам крутится импульсами,
строит ToF-панораму, релокализует курс по карте):

| Скан | Шаги | Воксели | Уверенных | reloc | Поправка курса |
|------|------|---------|-----------|-------|----------------|
| 70 с | 25   | 239     | 37        | 0     | — (порог 40 не достигнут) |
| 85 с | 29   | 247     | 45        | 1     | −0.8° |

**Узкие места (по данным):**

1. **Разрежённость ToF (главное).** Авто-профиль держит **4×4 = 16 зон**, из них
   валидны ~3–5 (остальные `-1`). Уверенные воксели (≥2 наблюдений) копятся
   медленно — за 70 с едва 37, релокализация (порог 40) не запустилась. 8×8 даёт
   **64 зоны = ~4× плотность** → быстрее насыщение уверенности.
2. **Лаг HTTP платы.** Цикл шага доходил до 2–4.5 с (таймауты `/tof`,`/telemetry`)
   — синхронный веб-сервер + поток MJPEG конкурируют. Лечение со стороны клиента:
   импульсный скан без камеры, короткие таймауты, пропуск шага.
3. **Грубая релокализация.** Старый матч — жёсткий «±1 воксель», шаг 1°: размытый
   пик на плоских стенах, точность ~1°.
4. **Хрупкость IMU.** Глитч смены батарейки подвесил MPU6050; `imu_ok` оставался 1
   при мёртвых нулях (`imu_temp` замер на 36.5 = сырой 0). Лечилось только ручным
   полным обесточиванием. Bias гиро калибровался лишь на старте (темп. дрейф за
   сессию не компенсировался).
5. **1-DOF.** Без энкодеров позиция x/z не растёт — режим turntable (только курс).
   Полный SLAM (позиция) — отдельный уровень (нужен SBC + энкодеры).

## 2. Исследование (методы оптимизации)

- **VL53L7CX 8×8 + sharpener.** Сенсор отдаёт матрицу 8×8 (64 зоны); **sharpener**
  (`set_sharpener_percent`, дефолт 5%, спец-значение даташита **14%** для 8×8)
  убирает «veiling glare» — растекание сигнала ярких краёв в соседние зоны → чище
  геометрия. Доступны per-zone **sigma/target_status** для фильтра качества.
  ([ST datasheet](https://www.st.com/resource/en/datasheet/vl53l7cx.pdf),
  [UM3038](https://www.st.com/resource/en/user_manual/um3038-a-guide-to-using-the-vl53l7cx-timeofflight-multizone-ranging-sensor-with-90-fov-stmicroelectronics.pdf))
- **Correlative scan matching (Olson, 2009) + likelihood field (Thrun,
  Probabilistic Robotics).** Вместо «попал/не попал» — **гладкое поле**:
  `occ_logodds · exp(-d²/2σ²)` от расстояния до занятого вокселя. Шире бассейн
  сходимости, суб-клеточная точность. **Coarse-to-fine** перебор для скорости,
  **low-pass блендинг** предсказания (IMU) и коррекции (CSM) для гладкости.
  ([Olson, real-time CSM](https://www.semanticscholar.org/paper/Real-time-correlative-scan-matching-Olson/f07e40a14e4f7e6d32bbaad32fe47cf61713e637),
  [CSM на occupancy grid](https://robomechjournal.springeropen.com/articles/10.1186/s40648-023-00256-w))
- **MPU6050 дрейф курса: ZUPT / «No-Motion-No-Integration».** Пауза интеграции на
  стоянке (deadband — уже было) + **пере-оценка bias на стоянке** (раньше только на
  старте; температурный дрейф нуля гиро накапливается).
  ([Yaw drift elimination](https://www.sciencedirect.com/science/article/abs/pii/S0924424721001540),
  [kriswiner/MPU6050](https://github.com/kriswiner/MPU6050))

## 3. Что сделано

**Сервер (PR #26, проверено `--reloctest`):**
- `WorldModel.likelihood_score` — likelihood-field скоринг (гладкое поле, мягкий
  порог: учитывает все наблюдённые воксели, не только confident → работает на
  более разрежённой карте).
- `relocalize_yaw` — **coarse-to-fine + параболическая интерполяция пика**.
  Точность восстановления дрейфа: **1° → 0.03°** (margin 0.98–1.00), берёт ±22°.

**Прошивка 1.4.1 (PR #27, ЗАШИТА И ПРОВЕРЕНА на железе 2026-06-16):**
- IMU: честный `imu_ok` (детект мёртвых нулей; `imu_temp` теперь живой ~31°) +
  **авто-реинициализация** при сбое + **ZUPT bias-tracker** на стоянке.
- ToF: **sharpener 14%** в 8×8; `/control?tofprofile=accurate` — рантайм-включение
  **8×8** (авто-профиль его не выбирает). Проверено: `res=8`, 64 зоны, **~2.5×
  валидных точек** против 4×4. `tools/spin_scan.py` запрашивает 8×8.

## 4. Результаты 8×8-скана на батарее (2026-06-16)

Полный скан с ездой (робот сам крутится), 8×8 + всё вышеперечисленное:

| Метрика | 4×4 | **8×8 + ZUPT** |
|---------|-----|----------------|
| Воксели | 247 | **732** (≈3×) |
| Уверенных | 45 | **248** (≈5.5×) |
| Релокализаций | 1 | **9** (сошлась, не убегает) |
| Курс | скакал | чистый, полный оборот |

Попутно вскрылись и закрыты две вещи:
- **Дрейф нуля гиро ~8°/с после авто-реинита.** При переходе USB→батарея глитч
  питания дёргает MPU6050; авто-реинит 1.4.1 поднимает сенсор, НО не пере-калибрует
  bias, а встроенный ZUPT-трекер (окно 0.8°/с) слишком узкий для большого дрейфа.
  **Фикс (сервер, `spin_scan.py`):** перед сканом мерим дрейф на стоянке и вычитаем
  линейно (bias ~постоянен за скан). Не нужен идеальный ребут. Правильный фикс —
  accel-based ZUPT в прошивке (см. ниже).
- **Лаг HTTP на батарее** — WiFi-эко-сон. Лечится `/control?wifi=1` (hi-power):
  ~10с/шаг → ~1.5с/шаг.
- **Показ карты без порчи:** `xiao_serial_telemetry.py --world-paused` грузит карту
  и НЕ вливает live-кадры (иначе дрейфующий гиро на стоянке «размазывает» карту).

## 5. Что осталось (следующие рычаги)

- **Прошивка: accel-based ZUPT** — на стоянке (|accel|≈1g) пере-оценивать bias гиро
  без узкого окна → дрейф само-исправляется после реинита/посреди скана, без ребута
  и без серверной компенсации. (Сейчас обходим в `spin_scan.py`.)
- **Фильтр точек по sigma/target_status** (per-zone) — экспортировать в `/tof` и
  отбрасывать шумные зоны на сервере (сейчас фильтр только по range_sigma в прошивке).
- **Позиция (полный SLAM)** — энкодеры + SBC (RTAB-Map); Wi-Fi-якорь (`wifi_anchor`)
  как грубый абсолютный фикс x/z. Это смена уровня, не тюнинг.
- **2-DOF релокализация** — когда появится трансляция, расширить `relocalize_yaw`
  до (yaw, x, z) тем же CSM (перебор/градиент по 3 параметрам).
