# Контракт ИИ-«мозга» ↔ плата

Мозг (`brain/`) **не требует изменений прошивки** — использует существующий
HTTP-контракт XIAO. Транспортно-нейтрален: тот же код запускается на
ПК-сервере или портируется на телефон (вариант «мозг на Android»).

## Входы (мозг читает с платы)

| Что | Запрос | Частота | Формат |
|-----|--------|---------|--------|
| Телеметрия | `GET /telemetry` | каждый цикл | JSON (см. ниже) |
| Кадр | `GET /capture` | каждый цикл (опц.) | один JPEG |

Используемые поля телеметрии: `drive_safety` (бит0 бампер, бит1 УЗ),
`bumper`, `us_cm` (0 = нет эха / вне диапазона — **не** препятствие),
`spd_l`/`spd_r`, `enc_l`/`enc_r`.

> `/capture` (один кадр) выбран вместо long-lived `/stream` ради простоты и
> надёжности контура ~4–5 Гц. MJPEG/низколатентный канал — опциональная
> будущая оптимизация (см. анализ esp32-cam-fpv), не блокирует архитектуру.

## Выходы (мозг управляет платой)

| Что | Запрос |
|-----|--------|
| Движение | `GET /drive?l=<-255..255>&r=<-255..255>` |
| Стоп | `GET /drive?stop=1` |

Если на плате включён `XIAO_API_TOKEN`, ко всем `/drive` и `/control`
добавляется `&t=<токен>` (флаг `--token`).

## Тайминги и безопасность

- **Темп команд:** `SafetyGovernor.command_interval_s` = `min(0.25,
  watchdog/2)` ≈ 0.225 с. Строго меньше watchdog прошивки (450 мс): в
  штатном движении робот не встаёт по watchdog, но если мозг умолк —
  встаёт сам (defense-in-depth).
- **Таймаут перцепции:** нет свежего «уверенного» решения дольше
  `perception_timeout_s` (1 с) → мозг шлёт стоп.
- **Рефлекс прошивки приоритетен:** при `drive_safety!=0`/`bumper!=0`
  governor немедленно отдаёт `(0,0)` — мозг не может «передавить» аппаратный
  рефлекс (ровно тот пробел, что виден у FOFOCA, здесь закрыт двумя слоями).
- **Потеря связи:** любая ошибка `/telemetry` или `/drive` → аварийный
  `stop()` + backoff. Контур всегда завершает работу `stop()` (finally).

## Перцепция (подключаемая)

Интерфейс `Recognizer.infer(frame_jpeg, telemetry) -> Intent(forward,
turn, confident)`. Реализации: `dummy` (без зависимостей, демо-цикл),
`brightness` (Pillow опц.). Реальная модель (YOLO/TFLite/внешний сервер)
добавляется новой реализацией `Recognizer` — архитектура от модели не зависит.

## Поток

```
/telemetry ─┐
/capture ───┤→ Recognizer.infer → Intent ─┐
            └────────────────────────────→ SafetyGovernor.decide → (l,r) → /drive
                                            (рефлексы > перцепция > движение)
```
